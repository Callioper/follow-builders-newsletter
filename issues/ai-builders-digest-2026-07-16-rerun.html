Boris Cherny（Anthropic Claude Code 的作者）聊 automation 为什么是这个时代最高杠杆的工程工作：「以前，最厉害的工程师会把大量时间花在把自己的工作自动化上——写更好的 vim/emacs 配置、写 lint rule、堆 e2e 测试。这些是工程师能做的最高杠杆的事。」

为什么现在更重要：「如果你在跑一支 agent 军，每个 agent 都会被这些 automation 加速。Automation 越多，单位时间的产出就越多。」Automation 不只是倍增你自己的产出——它倍增每一个将来会碰你 codebase 的 agent 的产出。

从一次性修复到 loop 的迁移：「你的 agent 每次看到这个问题都修一次，但那样要花 token，还可能漏 case。如果 Claude 改成写一条 lint rule、一个 CI step 或一个 routine，这一整类问题就永久自动化掉了。这就是大家说『loops』在指什么。」

更重要的转变：「Automation 让别人更容易给 codebase 贡献。我越来越多看到工程师入职第一天就能给 codebase 提 PR——因为 Claude 能替他们把 codebase 摸一遍——而非工程师现在也能像工程师一样贡献代码。」领域知识以前藏在人脑里；现在能被编码成 CLAUDE.md、REVIEW.md、skills 和代码注释。
