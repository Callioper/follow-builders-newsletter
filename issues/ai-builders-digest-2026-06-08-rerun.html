Theme 01

Loop 取代 prompt / Loops Eat Prompts

Steinberger 把「monthly reminder」讲成今年最狠的一条;Boris Cherny 接着把 Opus 的「长时跑」摊成 5 条 loop 原语;Sam Altman 一句「recursive loop」给整天的 vibe 命名;Peter Yang 用 dad joke 把 loop over prompt 收进家庭段子——单位工作正在从 prompt 移到 loop。

Peter Steinberger

Peter Steinberger 一条 13700+ 点赞的推,把「每月提醒」这一期说得比以往更狠:「别再 prompt 你的 coding agent 了,你应该设计让 agent 自己 prompt 自己的 loop。」

一句话的位移:单位工作以前是 prompt,现在正在变成 loop。prompt 是问题,loop 是岗位描述——可以跨 run 复用、可以 A/B、可以委托。然后 agent 退成「loop 里的执行者」,人退成「loop 的策展人」。

他的 bio 写「Polyagentmorphous ClawFather」,能跑并行 agent 的人。把这件事框成「monthly reminder」——也就是他至少说了一年——信号在于:这不是新看法,是一个已经定下来的 conviction。

Steinberger(13734L):别再 prompt coding agent 了,应该设计让 agent 自己 prompt 自己的 loop。单位工作从 prompt 移到 loop。

English

Peter Steinberger, in a tweet that pulled 13,700+ likes, makes this month's 'monthly reminder' land harder than usual: 'You shouldn't be prompting coding agents anymore. You should be designing loops that prompt your agents.'

The one-line shift: the unit of work used to be the prompt; the unit of work is becoming the loop. If the prompt is a question, the loop is a job description — durable across runs, easy to A/B, easy to delegate. The agent then becomes the executor inside a loop the human curates.

Steinberger's bio (Polyagentmorous ClawFather) tells you this is a man who runs a lot of agents in parallel. The fact that he frames the move as 'monthly reminder' — i.e. he's been saying it for at least a year — is the signal that this isn't a new take, it's a settled conviction.

Here's your monthly reminder that you shouldn't be prompting coding agents anymore. You should be designing loops that prompt your agents.

Claude Agent 平台的「dreaming」/ Claude's Self-Improving Agent Platform

Anthropic 把 dreaming、outcomes、multiagent orchestration 一起发,dreaming 是「session 之间反思 memory」的进程,outcomes 用独立 context 的 grader 评分——loop 在结构上就是产品,比标准 prompting loop 高 10 个点。

A
Anthropic
Anthropic 官方博客
中文

Anthropic 在 Claude Managed Agents 里发了「dreaming」的 research preview——一个周期性的进程,审看过去的 agent session,抽 pattern,curate memory,让 agent 自我改善。同一发版里还上了三个原语:dreaming(session 之间的 memory 精炼)、outcomes(按 rubric 自我批改,grader 跑在独立 context window)、multiagent orchestration(多 agent 协同)。

机制最值得读:outcomes 的 grader 跑在自己的 context window 里,「so it isn't influenced by the agent's reasoning」——这种「写一个,另一个打分,再回去改」的三段式,正好是 steipete 和 Cherny 讲的 loop。Anthropic 在把 loop 产品化:不再要求 agent 在原 context 里自我评估;loop 在结构上就是产品。

Anthropic 自己的数据:outcomes 比标准 prompting loop 在任务成功率上高 10 个点,难题上收益最大。可以这样读:2026 最有用的 loop 原语不是「更好的 prompt」,是「另一个 context 里的实例给你打分」。

Anthropic 推出 dreaming + outcomes + multiagent——outcomes 用独立 context 的 grader 评分,loop 在结构上就是产品,比标准 prompting loop 高 10 个点。

English

Anthropic ships a research preview of 'dreaming' in Claude Managed Agents — a scheduled process that reviews past agent sessions, extracts patterns, and curates memory so the agent self-improves. Three primitives land in the same release: dreaming (between-session memory refinement), outcomes (rubric-driven self-correction against a separate grader in its own context window), and multiagent orchestration (agents coordinating on a shared task).

The mechanism is what makes it interesting. The grader in 'outcomes' runs in its own context window, 'so it isn't influenced by the agent's reasoning.' That separation — agent writes, grader scores, agent rewrites — is exactly the loop that steipete and Cherny were describing. Anthropic is productizing the loop: the agent is no longer expected to evaluate its own work in-context; the loop is structurally the product.

Anthropic claims outcomes improved task success by up to 10 points over a standard prompting loop, with the largest gains on the hardest problems. Read this as: the most useful loop primitives in 2026 are not 'better prompts' — they're 'second instance in a separate context that scores you.'

Today we're launching dreaming in Claude Managed Agents as a research preview. Dreaming extends memory by reviewing past sessions to find patterns and help agents self-improve. We're also making outcomes, multiagent orchestration, and webhooks available to developers building with Managed Agents. Deliver better outcomes. With outcomes, you write a rubric describing what success looks like and the agent works toward it. A separate grader evaluates the output against your criteria in its own context window, so it isn't influenced by the agent's reasoning. When something isn't right, the grader pinpoints what needs to change and the agent takes another pass. In testing, outcomes improved task success by up to 10 points over a standard prompting loop, with the largest gains on the hardest problems.
