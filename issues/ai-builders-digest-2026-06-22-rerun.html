The harness gets eaten — Sakana Fugu 把 routing 挪进 model / The Harness Gets Eaten

Levie 给 Sakana 这一天收单:Fugu 发布的「multi-model」模式把 model selection / delegation / verification 全部挪进 model 自己,routing 不再是 harness 层的活。Levie 紧接一条把 agent 时代的尺度定在 100X——data leak / silent write / authoritative truth source / logging / verify 全部要重做。Steipete(OpenClaw 创始人)用一句「hunch was right」点头:作为跑过 multi-model agent 平台的人,他对 routing 应该归 model 自己的判断已经从怀疑变成确认。

Aaron Levie(Box CEO)给 Sakana 的 Fugu 收单:「又一个把 AI 架构往前推的新点子。Sakana 放出来的模型本质上是 models-of-models——一个 API 进去,work 被派给那个任务上最强的那个 model 去写。Fugu 自己管 model selection、delegation、verification、quality control。」

Levie:Sakana Fugu 把 multi-model routing 从 harness 层的活挪到 model 自己——「Fugu manages model selection, delegation, verification, and quality control」,下一个被压扁的是 model-selection graph。

English

Aaron Levie (Box CEO) on Sakana's Fugu release: 'Another new idea to push the state of AI architectures forward. Sakana released a model that effectively uses a mixture of models to get work done. You get a single API but then the work gets farmed out to the model that best performs the task. Fugu manages model selection, delegation, verification.'

Levie's editorial move is the inversion. Until now the 'routing' problem lived in the harness — your agent SDK, your orchestration code, your LangGraph-style graph picked the model. Fugu flips that: routing becomes the model's own job. A single API call returns the answer; the choice of which model wrote it is hidden behind the wrapper. This pushes 'multi-model routing' from a feature the harness exposes to a capability the model itself performs. If this pattern holds, the next layer to compress isn't the agent framework — it's the model-selection graph your harness maintains today.

Context > prompt — 输入大于输出才是质量信号 / Context Beats Prompt

Zhang Rui 这条是今年最可操作的质量指标。写「quick prompt」那种本能就是 AI slop 的来源。输入是输出 3-5 倍长,model 才有足够 context 做推理;输入比输出短,model 就是凭 vibe 现场编。对 builder-harness 设计,这是 gating 原则:任何让用户提交比输出短的 context 的产品,都是在配 slop。curated templates、结构化 intake、多源 context 拉取——这些是把 ratio 翻过来的产品动作。

zarazhangrui:防 AI slop 的经验是 input 比 output 长(常见 3-5 倍)——short input 就是 slop;harness 设计的 gating 原则是让输入大于输出。

English

Zhang Rui (Follow Builders founder / curator) on the AI-slop rule of thumb: 'A good rule of thumb for preventing AI slop (in writing, design, etc): Is your input (the context) longer than the output? I've found that for the AI to produce quality results, my input is often 3-5 times the length of the output. If your input is much shorter than your output, it's almost certainly slop.'

Zhang Rui's rule is the most actionable quality metric of the year. The instinct to write 'a quick prompt' is the AI-slop generator. When the input is 3-5x the output length, you're feeding enough context for the model to actually reason; when the input is shorter than the output, the model is improvising on vibes. For builder-harness design, this is the gating principle: any product that lets the user submit a shorter context than the output is configuring for slop. Curated templates, structured intake, multi-source context pulls — these are the product moves that flip the ratio.
