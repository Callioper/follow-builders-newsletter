Theme 01

Eval 是 applied layer 的真正瓶颈 / Evals Are the Real Bottleneck of the Applied Layer

Levie 把 2026 这年 applied-AI 的瓶颈从 model 转到 eval——open-weights post-training、agent improvements、agentic enterprise deployment 三件事的进度都挂在 eval 上;Box 当天放出 HTML preview / edit / version / share 的产品更新,把 agent-produced content 的 distribution 顺手接上,呼应同一条思路。

AL
Aaron Levie
Box CEO
@levie
中文

Aaron Levie(Box CEO)给 applied-AI 的进步来源做一次归因:「几乎所有 AI model 和 agent 的进步都挂在 eval 上。Open weights 在具体 domain 上 post-training 全靠 eval。Applied AI 层的 agent 改进全靠 eval。Agentic enterprise deployment 真能把工作接进去的,全靠 eval。全是 eval。」

Levie 这条真正的编辑动作是把瓶颈换个位置。在每家 lab 拼 model、每个 builder 拼 agent harness 的这一年,他把 throughput-limiting 的那一步挪到 eval suite——也就是那个告诉你「agent 在你这个具体 workflow 里到底做没做对」的系统。顺下来的推论是:能把自己(和客户的)workflow 画出来、能测 agent 在那个 workflow 里参与度的公司,「会处在真正推动 automation 的最佳位置」。Eval-as-core-competency 是 2026 的 enterprise moat thesis。

Levie:几乎所有 AI model 和 agent 的进步都挂在 eval 上——open-weights post-training、agent improvements、agentic enterprise deployment 三件事的进度全在 eval;能把 workflow 和 agent 参与度都量出来的公司,是 2026 真正能 push automation 的那批。

English

Aaron Levie (Box CEO) on where applied-AI progress actually comes from: "Almost all AI model and agent progress is downstream from evals. Open weights post training for specific domains comes down to evals. Agent improvements in the applied AI layer is all about evals. Agentic enterprise deployments that actually can augment work is all about evals. It's all evals."

Levie's editorial move is to relocate the bottleneck. In a year where every lab is racing on the model and every builder is racing on the agent harness, he claims the throughput-limiting step is now the eval suite - the system that tells you whether an agent actually did the work in your specific workflow. The downstream consequence: companies that can map their own (or their customers') workflows and measure agent participation in them "will be in the best position to actually drive real automation." Eval-as-core-competency is the new enterprise moat thesis for 2026.

Agentic enterprise deployments that actually can augment work is all about evals. It's all evals. This will become a core competency of any enterprise in the future. The companies that are able to best understand their own (and/or customers) workflows and how well agents participate in that work will be in the best position to actually drive real automation.

AI & I by Every - Mike Krieger on daily-driving Fable 5. Krieger (head of Anthropic Labs, Instagram co-founder) sits down with host Dan for ~50 minutes about what changes when you live with a mythos-class model for weeks instead of hours.

Three threads. (1) Day-one vs day-N: Krieger and host both reference the OPUS 4.5 / 4.6 December-into-January cycle, when extended users realized they hadn't pushed the previous model hard enough and had to rethink "what's even possible with this generation." Fable 5 is in that same window now. (2) Who has to learn what: inside Every, nontechnical knowledge workers come back saying "I need a new set of skills." People orchestrating agents come back saying "holy shit, there are so many new things I need to learn." The split is not capability - it's the shape of the workflow you're using the model inside. (3) Workflows get rewritten, not accelerated: Krieger's point is that you can't just run yesterday's workflow faster with Fable 5 - you have to redesign the workflow, because the model can now participate in steps it couldn't reach before. The applied-layer bottleneck Levie named in the morning is the same one Krieger is solving in the lab.
