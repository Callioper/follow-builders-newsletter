Claude 与 Apple 平台的原生接通 / Claude Lands Natively on Apple Platforms

Anthropic 同一天在 Claude Blog 发「Claude for Foundation Models」Swift 包,Apple 平台原生 agent 直接接通 Claude;AmandaAskell 顺手把 Claude 1 放回叙事;NotebookLM 同日升级跨源搜索 + 多格式输出。

C
Claude
Claude 官方博客
中文

Anthropic 在 Claude Blog 同日上线 Swift 包,把 Claude 接进 Apple 的 Foundation Models 框架。范式是:开发者用 Apple 端侧模型跑快、本地的小任务(摘要、抽取),需要「多步推理、代码生成等」的时候再把请求交给 Claude。

全篇最干净的一句:「Stream Claude's response back into the same view.」Foundation Models 框架靠 @Generable 注解返回类型化的 Swift 值,开发者到达 Claude 调用点时手里已经是结构化输入,不是原始用户文本。简单场景下,官方口径是三行代码。

产品读法:Anthropic 主动走到 Apple 开发者本来就在的地方——不是让每个 Swift dev 学 Claude 的 SDK,而是给一条用 Apple 自家框架当入口的路。「what this unlocks」那段是教科书式的扩展动作:日记 app 可以跨月找主题,学习 app 可以用「这件事和之前讲的有什么关系」追问,用户体验是一个,背后两个 model tier 各管各的。

Claude Blog:Foundation Models 框架接 Claude 的 Swift 包上线;端侧模型做快任务,Claude 接多步推理和代码生成,结构化输入输出,三行代码起步。

English

Anthropic's Claude Blog ships a Swift package that lets Apple's Foundation Models framework call Claude. The pattern: developers stay on Apple's on-device models for fast, local tasks (summarization, extraction), and hand off to Claude when the request 'calls for multi-step reasoning, code generation, and more.'

The cleanest line in the post: 'Stream Claude's response back into the same view.' The Foundation Models framework returns typed Swift values via @Generable annotations, so by the time the developer reaches the Claude call they have structured inputs, not raw user text. Three lines of code is the headline claim for the easy case.

The product read: Anthropic is meeting Apple where Apple's developers already are. Rather than asking every Swift dev to learn Claude's SDK, they ship a path that uses Apple's own framework as the entry point. The 'what this unlocks' framing is a textbook extension move — journaling apps can find cross-month themes, study apps can follow up with 'why does this matter for everything else we've covered,' all in one user-facing experience backed by two different model tiers.

Building intelligent apps for Apple platforms with Claude in the Foundation Models framework. Apple's Foundation Models framework gives developers access to tap into models natively from Swift. It is very easy to use and can return typed Swift values through guided generation in as few as three lines of code. Developers can use this to tap into Apple's on-device models for fast, local tasks like summarization or extraction. Developers can now use Apple's Foundation Models framework to hand off to Claude when a request calls for multi-step reasoning, code generation, and more. Stream Claude's response back into the same view.
