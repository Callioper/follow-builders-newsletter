Theme 01

Codex 越用越顺手 / Codex Gets More Useful by the Day

Sottiaux 把「/goal 自设」立成 meta-prompting 的一次推广——Codex 不只是听话执行,它自己也能 set task;Swyx 用 ultracode 体验给「subagent fanout」一种说法——子智能体不是更小的 worker,是 subroutines 的智能化版本,knowledge work 那些「剃完还有剃」的判断环节正是它最该去的地方。两条线合起来:harness 离 repo 越来越远,离 open web、离长期任务越来越近。

中文

Sottiaux(OpenAI Codex & ChatGPT)今天送上一个不大但很硬的推广:「Codex 能看、能设自己的 /goal。我们做的每样东西,都同时做成 agent 自己能用的工具。这是 meta-prompting 的一次推广——你让 agent 根据你的意图自己设任务。」位移很清晰:从「人 prompt agent」挪到「给 agent 一条路让它自己 prompt 自己」。

「我们做的每样东西都同时做成 agent 自己能用的工具」是更深的承诺。产品的 surface 不再只是用户;agent 自己是系统 primitive 的另一个用户。/goal 是这条线的具体落点,但架构层面的承诺是:任何 Codex 端的能力都得让 Codex 自己也读得懂。

Sottiaux:Codex 能看、能设自己的 /goal;产品同时是 agent 自己的工具,meta-prompting 的一次推广——让 agent 按你的意图自己设任务。

English

Sottiaux (Codex & ChatGPT at OpenAI) ships a small but real generalization: 'Codex can see and set its own /goal. Everything we build, we build also as a tool for the agent. This is a generalization of meta prompting, where you let the agent set its own task based on your intent.' The shift is from prompting the agent to giving the agent a way to prompt itself.

The framing matters: 'we build also as a tool for the agent' is the deeper claim. The product surface is no longer the user; the agent itself is a user of the system's primitives. /goal is a concrete instance, but the architectural commitment is that any Codex-side affordance has to be readable to Codex itself.

Codex can see and set its own /goal. Everything we build, we build also as a tool for the agent. This is a generalization of meta prompting, where you let the agent set its own task based on your intent.

learning loop 立成企业 AI 的正题 / The Learning Loop Becomes the Enterprise-AI Story

Amjad 转 Satya 那段「learning loop 是 IP,不是 frontier model」的长引文,在这周的分发边界画线里被两位 CEO 同时认领。Levie 把这条线讲完两层:模型层监管是 risk 的话,open weights 才是各国抢的赢家——监管挪到 model layer,反而把 sovereign AI 的 incentive 推到了别人手里;企业侧的正解是「把 unique IP / institutional knowledge 装进一个 learning architecture」,让 model 可以换、但「company veteran」跑不掉。

Amjad(Replit CEO)今天拎出 Satya 的引文,评价是「这周最让我受到鼓舞的 enterprise AI 的 positive-sum 视角」。他背书的那句话是:「真正的机会不是挑最好的 model,而是在 model 之上建一个 learning loop——让 human capital 和 token capital 都能复利。你可以外包一个 task,甚至一个 job,但你永远没法外包你的 learning。」

框架的位移比选 model 更重要。选最好的 model 被当成一次性决定;建一个 loop 才是耐用的投资。这条也是在做政治工作:Fable 分发事件占领时间线的这周,「positive-sum」是刻意选的词——潜台词是 enterprise AI 不是「谁拿到最强 model 谁就赢」的零和游戏,是「谁复利 learning 最快」谁就赢。

Amjad:背书 Satya「learning loop 才是 IP,不是 frontier model」——enterprise AI 是 positive-sum 比赛,比的是复利 learning 的速度,不是挑 model。

English

Amjad (Replit CEO) calls out a Satya quote that is doing real work this week: 'This is the most inspiring positive-sum vision for AI in the enterprise.' The line being endorsed: 'the real opportunity is not in picking the best model but instead in building a learning loop on top of models where human capital and token capital compound. You can offload a task, or even a job, but you can never offload your learning.'

The shift in the framing matters more than the model choice. Picking the best model is treated as a one-time decision; building a loop is treated as the durable investment. The post is also doing political work: when the Fable distribution story is dominating the timeline, 'positive-sum' is a deliberate choice — the implication is that enterprise AI is not a zero-sum race for whoever owns the strongest model, it's a race for whoever compounds the learning fastest.

This is the most inspiring positive-sum vision for AI in the enterprise.

English

The claim is that the scarce skill has shifted from running a model to running a workflow of agents.
