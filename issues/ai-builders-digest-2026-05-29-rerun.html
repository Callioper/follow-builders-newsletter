Theme 02

Context Is the New Bottleneck / 上下文成为新瓶颈

Levie 把企业 AI 的第一问题点得很清楚：knowledge work agent 真正的难点不是模型，是上下文。

AL
Aaron Levie
Box CEO
@levie
中文

Aaron Levie 直接把上下文问题叫做「企业 AI 真正的头号问题」。Agentic coding 之所以没爆雷，是因为代码库本身就是一个巨大的上下文文件，而且用户也够技术，可以把剩下的部分喂给 agent。

知识工作 agent 没这个条件。Box 的客户每天都在撞这堵墙：现有的数字知识分散在老系统里、跟 agent 兼容性差、权限控制对不上真实工作，而最关键的上下文还藏在人脑里的 tribal knowledge，必须先被数字化成非结构化数据。

他对这件事的判断：这是 applied AI 公司最大的杠杆点之一——谁能帮 agent 拿到正确的信息，谁就赢。这也是为什么 FDE 和新一代系统集成商会活得很好的原因。能解决这个问题的公司，会把其他人拉开一个身位。

Levie：上下文是 enterprise AI 的头号问题。

代码库本身是大上下文，用户也够技术——knowledge work 没这个条件。

知识分散在老系统里，最关键的还藏在人脑里，要先数字化才能喂给 agent。这是 FDE 和系统集成商的机会。

English

Aaron Levie calls the context problem effectively the number one problem for AI agents in the enterprise. Agentic coding got away with it because the codebase is itself a giant context file, and the user is technical enough to hand the rest to the agent.

Knowledge work agents do not have that luxury. Box customers are hitting it every day: existing digital knowledge is fragmented across legacy systems that do not play nice with agents, access controls do not map to real work, and critical context is still living in people's heads as tribal knowledge that has to be digitized into unstructured data first.

His take on what this means: it is one of the biggest points of leverage for applied AI companies that specialize in getting agents the right information. It is also why FDEs and new system-integrator plays are going to do so well right now. The companies that figure this out will pull away from the rest.

This is effectively the #1 problem for AI agents in the enterprise. As we go from agentic coding (where a large amount of context is in the code base, and users are technical enough to get the rest to the agent easily) to a world of knowledge work agents, the context problem becomes much more acute.

We see this every day with customers at Box. For existing digital knowledge, it's often fragmented across legacy systems or environments that don't play nice with agents, and have access controls that don't map to the real work that needs to be done.

But also, companies often haven't captured and digitized some of the critical context that agents need to work with. Decisions, processes, and workflows often live in people's heads and tribal knowledge that need to get turned into unstructured data for agents.
