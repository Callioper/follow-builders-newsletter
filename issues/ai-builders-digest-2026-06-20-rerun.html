Agent 的「上下文 + 共享工作区」是真正瓶颈 / Context and the Shared Workspace

Levie 把这一天真正决定成败的那一层指出来——不是 model、不是 prompt、不是 harness,而是 agent 读写时所在的工作区,以及人能不能看着同一片工作区就明白 agent 在干什么;Peter Yang 公开从 Claude Code 倒戈 Codex,理由列得清清楚楚(模型、unit 经济学、UX、browser / computer use 底线);Boris Cherny 用同一套 harness 解 3500 年前的克里特线性 A,证明 Claude Code 已经不只是 coding 工具,是一张能托住任何分析型任务的工作面。

AL
Aaron Levie
Box CEO
@levie
中文

Aaron Levie(Box CEO)讲真正决定 agent 成败的那一层:「让 agent 真的能跑出来的最大变量,是能不能把完成任务需要的 context 喂到它面前;而这件事里很关键的一条,是能不能给 agent 弄出一个**人也能看懂**的共享工作区。这也是为什么用文件系统的 agent 跑得顺。」

Levie 在点的就是那层真正的扛把子——不是 model、不是 prompt、不是 harness,而是 agent 读写时所在的那个工作区,以及人能不能看着同一片工作区就明白 agent 在干什么。文件系统之所以赢,是因为它同时满足「人看得懂」和「agent 读得动」两条;聊天记录两面都不沾。

Levie:agent 成不成,真变量是「context 喂不喂得到」+「共享工作区人看不看得懂」——文件系统的胜利就在这两条。

English

Aaron Levie (Box CEO) on what actually determines agent success: 'The main variable in getting success with agents is whether you can get the agent the context it needs to do its work; and a major factor in that is if you can create a shared working area for that agent that a human can understand as well. This is one of the reasons why agents using file systems work so well.'

Levie is naming the load-bearing layer. It is not the model, not the prompt, not the harness — it is the workspace the agent reads and writes inside, and whether a human can look at the same workspace and understand what the agent is doing. The file system is winning because it satisfies both halves of that constraint; a chat log satisfies neither.

The main variable in getting success with agents is whether you can get the agent the context it needs to do its work; and a major factor in that is if you can create a shared working area for that agent that a human can understand as well. This is one of the reasons why agents using file systems work so well.
