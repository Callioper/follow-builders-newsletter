100x inference 的应用层逻辑 / Where 100x Inference Actually Comes From

Levie 把 Devin 的 agentic mapreduce 立成「为什么我们需要 100x more AI inference」的具体答案——swarms of agents + bounded shards + reduce + isolated sandbox verification,这套 pattern 在 code security 之外会被复制到 pharma / banking / 所有 unstructured data。

AL
Aaron Levie
Box CEO
@levie
中文

Aaron Levie(Box CEO)用 Devin 的 agentic mapreduce 答「为什么我们要 100x more AI inference」:Devin 跨 repo 找 signal,fans out focused agents over bounded shards,reduce 到 isolated sandbox 做 verification。code security 是 first case,pharma / banking / 所有 unstructured data 都会被这套 pattern 复制。

Aaron Levie:如果你曾好奇「为什么未来我们要 100x more AI inference,这件事是被什么驱动的」——Devin 又给了一个具体答案。Devin 提出 agentic mapreduce 这个想法:会有 swarms of agents 在大量数据(代码)上跑,处理人以前根本做不到的任务。「Devin 在 repo 里 map 出相关 signal,把 focused agents fan out 到有界的 shard 上,把它们的 findings reduce 到一个 report,然后在 isolated sandbox 里 verify 严重漏洞再标 confirmed。」这一例是 code security,但 code 和 knowledge work 里有大量其他 use case。Box 这边看到的客户想处理和理解 millions of 文档,做 risk、insight、关系等。这一 pattern 会在 pharma、banking 和所有 unstructured data 上铺开。顺带一句,这类能力通常只在你能 deploy 一组模型(frontier 加 lower cost 都要)时才做得到,因为 token 量级巨大。这会是 applied AI layer 的一个主要价值主张。

English

Aaron Levie(Box CEO)用 Devin 的 agentic mapreduce 回答「为什么我们需要 100x more AI inference」:Devin 跨 repo 找相关 signal,fans out focused agents over bounded shards,reduce 之后到 isolated sandbox 做 verification——这种 pattern 在 code security 之外会被复制到 pharma / banking / 所有 unstructured data。

If you've ever wondered why we will need 100X more AI inference in the future, and what it's going to be driven by, this is another good example. Devin pushes forward an idea of agentic mapreduce, which means we'll now have swarms of agents that are processing large amounts of data (code) to handle tasks that humans never could have done before. "Devin maps relevant signals across the repo, fans out focused agents over bounded shards, reduces their findings into one report, then verifies serious vulnerabilities in isolated sandboxes before marking them confirmed." In this case it's code security, but there are tons of other use-cases in code and knowledge work. We see this at Box with customers that want to process and understand millions of documents for risk, insights, relationships, and more. This will play out in pharma, banking, and many other industries across all forms of unstructured data. As an aside, these types of capabilities are generally only possible when you can deploy a variety of models (both the frontier and lower cost) because of the sheer amount of tokens that go into these use-cases. This is going to be a major value proposition for the applied AI layer.
