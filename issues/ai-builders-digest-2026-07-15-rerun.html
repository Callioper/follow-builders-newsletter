Theme 01

Training Data：Anthropic Platform 团队讲 coordination + ecosystem 不是 walled garden / Training Data: Anthropic Platform on Coordination Layers and an Open Ecosystem

Anthropic Platform 团队（Katelyn Lesse + Angela Jiang）这一期把下一层抽象讲清楚了：knowledge → execution → coordination，harness 要从 raw execution 变成 meta-harness 把 strategies 编排起来；同一条线把「ecosystem 不是 walled garden」的哲学展开——self-hosted sandboxes 跟 Modal/Vercel/Cloudflare/Amazon micro VM 全做了 first-class、MCP tunnels 穿透防火墙、skills / MCP 直接当 standards 给出去。

TD
Training Data Podcast
AI 播客
中文

Anthropic Platform 团队（Katelyn Lesse + Angela Jiang）这周做客 Training Data，把平台下一层抽象讲清楚了：coordination layer——strategies 当 meta-harness 架在 execution 之上。

Katelyn Lesse 给的抽象 ladder 是「knowledge → execution → coordination」。现在这版低层 harness 还在做执行层面的事，下一版要解决的是把 orchestrated strategies 拼起来——同一种 token 拿不同 job，有的当 advisor，有的当 executor，原语一样、分工不一样。

Katelyn 把 roadmap 讲得很直：「你会看到我们从 execution 层一步步推到 coordination 层，抽象越堆越上面。」对 builder 的含义是：harness 接下来变 meta，不是变 raw。

在这一切之上的最后一层抽象，大概是 coordination layer。所以有 knowledge、有 execution、有 coordination。

它差不多是一个 meta-harness。真正的低层 harness 是为执行设计的，但下一层要解决的是：token 之间不完全 fungible，你需要给它们分配不同的工作——这个 token 在做 advisor，那个 token 在做 executor。

如果你看我们的 roadmap，再往前走一点，你会看到我们的抽象会从 execution 层一步步推向 coordination 层，这是你能看到我们放出来的东西。

English

Anthropic Platform team (Katelyn Lesse + Angela Jiang) on this week's Training Data: the platform's next abstraction is the coordination layer — strategies as a meta-harness sitting on top of execution.

Katelyn Lesse frames the abstraction ladder: 'knowledge, execution, coordination.' The current low-level harness is built for execution, but the next one is about composing orchestrated strategies where some tokens advise and others execute — same primitives, different jobs.

Katelyn's road map claim: 'we'll move more and more from the execution layer to the kind of coordination layer in terms of the abstractions that you can see us put out.' The implication for builders is that the harness is about to get more meta, not more raw.

The last layer of abstraction on top of this is probably the coordination layer. So you have knowledge, and you have execution, and you have coordination.

It's almost like a meta harness. The true low level harness is designed for execution. But the next one is about, okay, if tokens aren't really fungible and you need to give them different jobs, like maybe this token is advising versus this token is executing.

If you were to look at our road map and the maybe kind of project forward a little bit where you kind of expect us to go, we'll move more and more from the execution layer to the kind of coordination layer in terms of the abstractions that you can see us put out.
