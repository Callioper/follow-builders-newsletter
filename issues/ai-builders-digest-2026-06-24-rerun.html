Boris Cherny(Anthropic,Claude Code 作者)给 Claude Tag 画出运行时形态:在 channel 里 @ Claude,它就拉起自己的 instance 和 sandbox——clone repo、写代码、跑 test、build,全在那块隔离环境里做,做完 sandbox 就扔。一个 thread 一个 instance,每个 channel 有自己的 memory 和权限。

「一个 thread 一个 instance」是这次的结构性决定。意思是每次 tag 都是一次 bounded scope 的新执行;跨 channel 的 context 存在另一层 memory 里,不在 sandbox 里。这也是为什么 ambient 主动模式在嘈杂的工作空间里能放心打开:每次 prompt 是隔离的,memory 是 per-channel 的,surface 是可审计的。

Tag Claude in a channel, it spins up an instance with its own sandbox. It clones repos, writes code, tests, compiles all in that isolated environment and the sandbox gets thrown away when it's done. One instance per thread, its own memory and permissions per channel.

English

Boris Cherny (Anthropic, the creator of Claude Code) on the runtime shape of Claude Tag: you tag Claude in a channel, it spins up its own instance with its own sandbox. It clones repos, writes code, tests, compiles — all in that isolated environment — and the sandbox is thrown away when it's done. One instance per thread, its own memory and permissions per channel.

The 'one instance per thread' choice is the structural decision. It means every tag-claude is a fresh execution with bounded scope; the cross-channel context lives in a separate memory layer, not in the sandbox. That's what makes the proactive ambient mode safe to turn on in a busy workspace: each prompt is isolated, the memory is per-channel, and the surface is auditable.

Ask Claude to react to the top-level thread with ⏲️ ✅ ❓ 🛑 — ⏲️ while it's working, ✅ when done, ❓ when blocked on a question, 🛑 when it can't proceed. The interesting move is treating emoji reactions as a status channel. The reaction API is the one piece of state Slack exposes that's both human-readable and machine-writable from a per-thread agent instance — which makes it the natural surface for agent-to-human status reporting in a multi-player setting.

Andrej Karpathy 把 Claude Tag 框成 LLM UI/UX 的第三次大重设计。第一次范式:LLM 是一个你打开的网站;第二次:它是你下载的 app;第三次——也就是 Claude Tag 在做的——是一个自带 org 范围 tools 和 context、持续在线、异步运行、跟人类团队并行工作的独立实体。

Karpathy 的「just works」一句点到这次最难的:把这种设计做到「just work」,要在 org 层面把 tools、integrations、计算环境、memory、security 一起解决,不是发一个 chat 框就完事。这些都到位之后,交互的感觉就变成「跟一个能接很多种活儿的队友说话」,不是「用一个工具」。

This is a new paradigm for interacting with Claude that is significantly more "inline" with all the other human activity org-wide. Once you do all the under the hood engineering work to make this "just work" (e.g. across tools, integrations, compute environments, memory, security, etc.), Claude basically joins the team in a seamless way - you can talk to it as you would talk to a person and it can help with a very large variety of workloads. Imo this is the 3rd major redesign of LLM UIUX. The first paradigm was that the LLM is a website you go to, the second was that it is an app you download to your computer. This third one is that it is a self-contained, persistent, asynchronous entity with org-wide tools and context, working alongside teams of humans.

English

Andrej Karpathy frames Claude Tag as the third major redesign of LLM UI/UX. The first paradigm: the LLM is a website you go to. The second: it's an app you download. The third — what Claude Tag is shipping — is a self-contained, persistent, asynchronous entity with org-wide tools and context, working alongside teams of humans.

Karpathy's 'just works' framing points at the hard part: making this design 'just work' requires solving tools, integrations, compute environments, memory, and security at the org level — not just shipping a chat box. Once those are in place, the interaction feels less like 'use a tool' and more like 'talk to a teammate who can take on a large variety of workloads'.
