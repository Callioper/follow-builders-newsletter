模型路由层开始值钱 / The Model-Router Layer Becomes Valuable

Levie 把「AI 模型路由到最好的那个」单独立成一层价值主张——成本优化、可避免的失败模式、和未来模型变多时单一模型不可能同时赢所有任务。三条原因加起来,routing 这层比单一模型订阅更值得付钱。

AL
Aaron Levie
Box CEO
@levie
中文

Aaron Levie(Box CEO)谈 routing 层开始变贵:「能把请求路由到「做这个任务最好的那个模型」的层,价值会大幅上升。至少三个原因:成本优化(部分任务要 frontier,其它要便宜的)、失败模式规避(不同模型失败方式不同,挑对的就行)、以及模型能力摊开之后,没有单一模型能赢所有任务。」

Levie 的框是结构性的:routing 层不是 wrapper,是业务逻辑居住的地方。「为任务挑模型」这个决定编码了应用真正在乎的东西——能接受哪种失败、latency 预算多少、单次请求的成本上限。应用把这套策略写一次,router 在不断变长的 model catalog 上强制执行它。

Levie:为任务挑模型的 routing 层变得值钱——成本、失败模式、模型多样性三个原因叠加,这是一个业务逻辑居住的层,不只是 wrapper。

English

Aaron Levie (Box CEO) on the routing layer becoming a valuable product surface: 'The layer that can route to the best AI model for the particular job is going to increase in value substantially. There are at least 3 big reasons: cost optimization (frontier for some tasks, cheaper for others), failure-mode avoidance (different models fail in different ways, so picking the right one for the task matters), and the fact that as model capability spreads, no single model wins every task.'

Levie's framing is structural: a routing layer is not a wrapper, it's a place where business logic lives. The pick-the-model-for-the-task decision encodes what the application actually cares about — which kind of failure is acceptable, what latency budget is, what cost ceiling is per request. The application writes that policy once; the router enforces it across an ever-changing model catalog.

Theme 05

品味与新地图 / Taste and New Maps

Zhang Rui 转 Paul Graham 的「taste」长文并把「mastery + experience」拉成 tldr;Garry Tan 两帖把现状框成「拿着旧地图走新地形」——AI 时代需要重新画地图,方法是走一遍。两条线最后都回到同一句话:对所做的事的判断力比答案稀缺。

ZR
Zhang Rui
Follow Builders 创始人 / 策展人
@zarazhangrui
中文
