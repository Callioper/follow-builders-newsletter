English

Aaron Levie (Box CEO) on the open-weights shift: 'Pretty remarkable what's happening with open weights AI right now. We're seeing models achieve SOTA results on specific tasks, and getting close to frontier on some areas of coding and other domains. Levie's editorial move is in the second half — frontier labs are not threatened by open weights closing the gap, they are relieved. The applied layer is the one that gets to cost-optimize: cheaper models for bulk work, fine-tuned open models for specific tasks, frontier models for everything that has to be right.

Incidentally, this is actually fine for the frontier labs as well; if we can lower the cost of an overall task then AI usage goes up in general. You're still likely using frontier models for planning, orchestration, reviewing, and other parts of work. But this is all very good for the applied layer of AI, which is now in a great position to cost optimize workloads with cheaper models or use tailored open models post-trained for specific tasks to improve performance.

English

Anthropic Engineering publishes 'Scaling Managed Agents: Decoupling the brain from the hands' — the architectural companion to the Claude Code postmortem.

Zhang Rui(前端 / 设计 founder)点名一个写不进 system prompt 的难题:「Proactiveness 听起来很好听,做起来真的难。」

Proactiveness 是每个 agent 的 system prompt 都会写一句的特性。Zhang Rui 在说为什么难做:proactiveness 是判断题——什么要主动提、什么时候升级、什么时候闭嘴。这件事没有训练信号,而且 agent 没有跟用户共享的「用户当下在想什么」上下文。这不是「agent 太被动」或「agent 太冒进」——是校准这件事需要的上下文,agent 没有。这是 agentic 产品这一代没解决的 UI 难题之一。

Zhang Rui:Proactiveness 听起来好,做起来真难——是判断题,没有训练信号,agent 没有跟用户共享的上下文。

English

Zhang Rui (frontend/design founder) on the silent hard problem: 'Proactiveness sounds nice in theory but is so hard to get right.'

Proactiveness is the AI-agent trait everyone writes into their system prompts. Zhang Rui is naming why it is hard in practice: proactiveness is about judgment — what to bring up unprompted, when to escalate, when to stay quiet. There is no training signal for that, and the agent has no shared context with the user about what the user is already thinking about. The complaint is not 'agents are too passive' or 'agents are too aggressive' — it is that the calibration requires a level of context the agent does not have. This is one of the unsolved UI problems of agentic products.

Proactiveness sounds nice in theory but is so hard to get right
