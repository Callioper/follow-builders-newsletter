Vibe coding 当成工艺 / Vibe Coding as a Craft

Nan Yu 给出 agent UX 的工艺级复盘——one-shot 让用户跳过思考,多轮让 agent 反过来问「要强调什么」才是质量控制;Zara Zhang 的 Frontend Slides skill 让 skill 开始变成 taste 的最小复用单元。

NY
Nan Yu
AI 创业者 / 投资人
@thenanyu
中文

Nan Yu(Linear 产品)给出一份 agent UX 的工艺级复盘:「这个 pattern 我们调了好几轮才定下来。早期我们太天真,以为 agent 能 one-shot 写出项目更新,几乎不用用户参与。结果产出一堆 slop,因为用户在写更新的过程中把脑子完全脱开了。改成多轮交互模式——agent 反过来问你要强调什么、最重要的点是什么、缺什么上下文——之后,这个简单的改动拉出了明显更好的项目更新,也保证了用户始终有机会 steering,而不是只被要求 approve。」

Nan 的复盘是「agent 当 collaborator,不当 typist」这个设计最干净的论据。demo 里 one-shot 看着漂亮,生产里产 slop,因为它让用户跳过了思考。修法——多轮、agent 反过来问你强调什么——不是 UX 细节,是质量控制机制。用户的脑子留在线上,是因为 agent 逼着它留在线上。

Nan Yu:agent 一次性产出项目更新看起来爽,实际产 slop——因为用户跳过了思考。多轮 + 让 agent 反过来问「要强调什么」,才是 quality control 机制。

English

Nan Yu (Linear product) shares a craft-level lesson on agent UX: 'It took us a lot of iterations to find the right pattern for this. Early on we were very naive and we thought that the agent could one shot project updates without much input from users. That approach ended up creating a bunch of slop, because users disengaged their brain from the process of writing an update. By switching to a multi turn interactive mode, the agent prompts you and asks you what to emphasize, what's most important, what context is missing, etc. This simple change elicits much better project updates and ensures that the user has a chance to steer.'

Nan's post-mortem is the cleanest argument for the 'agent-as-collaborator, not agent-as-typist' design. One-shot generation sounds appealing in the demo; in production it produces slop because it lets the user skip the thinking. The fix — multi-turn, where the agent asks you what to emphasize — is not a UX polish, it is a quality control mechanism. The user's brain stays in the loop because the agent forces it to.

It took us a lot of iterations to find the right pattern for this. Early on we were very naive and we thought that the agent could one shot project updates without much input from users. That approach ended up creating a bunch of slop, because users disengaged their brain from the process of writing an update. By switching to a multi turn interactive mode, the agent prompts you and asks you what to emphasize, what's most important, what context is missing, etc. This simple change elicits much better project updates and ensures that the user has a chance to steer rather than simply being asked to approve something.
